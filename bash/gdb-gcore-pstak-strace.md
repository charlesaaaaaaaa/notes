# gdb 是用来调试进程的，
## `gdb -p 进程号` 是用来打印变量的值的。
### 如果一个进程出现了问题，这个时候就可以用`lsof -i:端口号`查找出对应的pid
### 再使用`gdb -p pid`就可以直接调试这个进程,再使用`bt`命令查看函数堆栈
```
list     : 简写l
       列出当前源码，“+”和“-”分别表示像前、向后列出源码。

break    ：简写b
       设置断点。可以用 “info break”查看断点信息。

next     ：简写n
       单条语句执行。

continue :简写c
       继续程序执行，遇到断点停止，否则执行到程序结束。

print    ： 简写 p
       打印变量的值。

backtrace： 简写bt
       查看函数堆栈。

finish   ：
      退出函数。
```
